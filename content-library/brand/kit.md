# Brand kit: GTM Tech Week (visual)

The minimum needed to make a dashboard, slide or asset look like us. Pairs with `voice.md` (the words).

## Palette
| Role | Hex | Use |
|---|---|---|
| Off-white (paper) | `#fefaef` | Default background. Everything sits on this, not on white. |
| Black (ink) | `#000000` | Body text, wordmark, rules. High contrast, no soft greys for type. |
| Acid green (signal) | `#b8ff03` | The accent. Highlights, key numbers, one CTA per screen. Use sparingly. |
| Magenta (alt signal) | `#fc3dd5` | Secondary accent. Pair with acid green for the two-tone look; never use both as fills on the same element. |

Rules:
- Background is **off-white `#fefaef`**, not white. This is the single biggest "looks like us" cue.
- Acid green is a **signal colour**, not a fill colour. If half the screen is green, it is wrong.
- Black type on off-white for everything readable. No grey body text.

## Type
- **Headlines:** a tight grotesque / neo-grotesque sans (Inter, Neue Haas, or similar), heavy weight,
  sentence case, negative tracking on big sizes.
- **Body:** the same family, regular weight, generous line height.
- **Numbers and labels:** can go UPPERCASE for eyebrows and data labels only. Never title case in
  running copy.
- **Mono** (for data, ticket counts, code-ish UI): a plain monospace is fine for dashboards.

## Layout
- Lots of off-white space. Editorial, not busy.
- One acid-green accent per view. One idea per slide.
- Left-aligned. No centred paragraphs.

## For a brand-styled dashboard (the UC finale)
Background `#fefaef`, black type, the headline KPI in acid green, a single magenta accent for the
secondary metric. Keep it flat. The point is "make our data look like us," not chartjunk.
